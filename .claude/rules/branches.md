# Branch Naming

Format: `<type>/<short-description>`

| Type | Usage |
|------|-------|
| `feat` | New feature |
| `fix` | Bug fix |
| `refactor` | Code restructuring |
| `chore` | Tooling, config, dependencies |
| `docs` | Documentation only |
| `test` | Adding or updating tests |

Examples:
- `feat/morning-brief`
- `fix/healthkit-sync`
- `refactor/api-routes`
- `chore/ci-setup`
- `docs/api-reference`
